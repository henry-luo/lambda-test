# test262 Non-Batch List
# Tests that MUST run individually (batch size = 1), never in a shared batch.
#
# These tests destructively mutate global built-in state (prototype overrides,
# property deletions, etc.) without restoration. In a shared batch process,
# such mutations leak into subsequent tests, causing spurious failures.
#
# In a spec-compliant test262 runner each test gets a fresh JS realm. Our batch
# mode shares one process for ~50 tests, so we quarantine known poisoners here.
#
# To identify new poisoners: run with --batch-only, check Phase 4 recovered
# tests in temp/_t262_batch_kills.txt, then binary-search the batch to find
# which preceding test causes the failure.
#
# Format: one test_name per line (same as baseline file).  # comments allowed.
